00247 /*!
00248  * @brief Finds a single resource for a given identifier using the default URL.
00249  *
00250  * @par Ruby on Rails Comparison
00251  * Under the Rails' ActiveResource gem, this method appears as a private
00252  * method. Why not here? In Rails, you access the @c find_single(scope, options)
00253  * method indirectly as the default @c find(arguments) case when the first scope
00254  * argument does not match all, first, last or one (by symbol). Objective-C does
00255  * not offer so flexible a syntax. Consequently, this implementation folds the
00256  * find-scope interface into distinct methods: find all, find first, find last,
00257  * find one and find single. The scope argument resolves to the method @e
00258  * name. This approach carries advantages and disadvantages. It eliminates the
00259  * @c switch statement necessary to resolve the scope. But at the same stroke
00260  * eliminates the flexibility of parameterising the scope in cases where scope
00261  * is a dynamic argument.
