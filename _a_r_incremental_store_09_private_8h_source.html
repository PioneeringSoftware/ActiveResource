00054  * @brief Answers a 64-bit version number derived from the given @a resource.
00055  * @details Uses the updated-at date-time as the version number. Converts the
00056  * update-at date to seconds since the reference date. This amounts to a big
00057  * version number, but the version number allows for 64 bits of unsigned integer

00059  *
00060  * Also makes an assumption about the updated-at dates: that they always exceed
00061  * midnight 1st January 2001, the reference point. If they precede that point,
00062  * then the reference-relative time interval becomes negative, the signed
00063  * interval wraps the unsigned 64-bit version and version numbers start counting
00064  * down. This is not what Core Data will expect.
00065  *
00066  * The implementation multiplies the time interval by 1,000 in order to allow
00067  * for rare sub-second updates, if the server-side includes date-time at
00068  * sub-second resolutions. RFC 3339 date-time formats allow for sub-second
00069  * accuracy.
